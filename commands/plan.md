# /plan

Apply the `feature-planning` skill to the request that follows (or to the current task if none is given).

Before proposing any implementation:
1. Restate the goal and acceptance criteria.
2. Search the codebase for existing components, hooks, utilities, and data-fetching patterns to reuse.
3. List the concrete files to add or change.
4. Define the test + types + UI plan.
5. Surface risks and open questions, and ask about anything ambiguous or irreversible before proceeding.

Do not write implementation code in this step. Output the plan and wait for confirmation.

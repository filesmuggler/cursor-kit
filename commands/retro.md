# /retro

Apply the `retro` skill to the current session.

1. Summarize what the session aimed to do and the outcome.
2. List what worked well.
3. List friction, mistakes, and rework, with root causes.
4. Map each lesson to a concrete change in `rules/`, `skills/`, or `commands/`.
5. Propose the edits as minimal, high-signal diffs for the user to accept.

Only encode patterns that recurred or caused real cost. Do not over-engineer.
